공유기(router)

IP address
다른 컴퓨터와 통신하기 위한 컴퓨터의 주소값
LAN
근거리 통신망, 로컬 영역 네트워크(영어: local area network, LAN), 구내 정보 통신망은 네트워크 매체를 이용하여 집, 사무실, 학교 등의 건물과 같은 가까운 지역을 한데 묶는 컴퓨터 네트워크이다.
WAN
광역 통신망(영어: wide area network, WAN)은 드넓은 지리적 거리/장소를 넘나드는 통신 네트워크 또는 컴퓨터 네트워크이다. 광역 통신망은 종종 전용선과 함께 구성된다.
각각의 통신장비들은 공유기에 연결되는 순간 각각의 IP주소를 부여받는다.(private IP adress 사설IP) 공유기 또한 고유의 IP주소를 부여받는다.(gateway address-LAN)(public IP address 공용IP-WAN)

NAT(Network Address Translation)

NAT(Network Address Translation)
외부로(WAN) 정보를 요청->공유기 안에 요청한 컴퓨터(192.168.0.4)의 정보를 저장-> NAT이 요청한 데이터를 WAN(59.6.66.238)로 변경하고 변경된 데이터를 위키피디아(외부 정보)로 전송-> 위키피디아는 WAN 공유기로 다시 전송-> 컴퓨터(192.168.0.4)로 전송 => 쉽게 말해 LAN과 WAN의 연결 역할을 한다.

포트(port)

외부에서 요청이 들어왔을 때 공유기에 연결된 컴퓨터중 누구를 서버로 사용할 것인지 연결에 줄 수 있도록 하는 방법

포트(port)
  • 하나의 컴퓨터에는 웹 서버가 여러가지 존재할 수 있음. 그 서버들을 식별해야 함.
  • 80-http 웹은 http라는 프로토콜을 이용하고 있는 통신 시스템. 80번은 웹이 쓰도록 정해진 규약. 컴퓨터에 웹 서버를 설치하면 기본적으로 80번 포트를 사용(listening)하도록 약속되어 있음.
  • 만약 웹 서버를 하나 더 설치하려 하면 8080 포트(http://dhchun1203.org 8080)를 사용(관습적)
  • 0 / 22-SSH / 80-http(http://dhchun1203.org) / 1023 : Well-known port
  • Well known port 에 들어가는 웹 사이트를 개발하면 성공한 인생^^

포트 포워딩(port forwarding)

외부 사용자들은 공유기 WAN 포트번호로 접속 -> 이제 공유기에 포트 포워딩을 설정(WAN[59.6.66.238:8080]) => 192.168.0.4:80)

만약 서버가 2개라면 추가로 설정해주면 됨.(WAN[59.6.66.238:8080]) => 192.168.0.3:80) #라우터는 일종의 안내자 역할을 함.

주소창에 라우터의 IP주소를 치면 공유기 설정창이 나옴. -> 고급설정 -> NAT/라우터 관리 -> 포트포워드 설정 -> 내부 IP주소 192.168.0.4 입력 -> 외부 포트 8081입력 -> 내부 포트 80입력 -> 적용 -> my ip 검색 -> 주소 창에 공유기의 IP입력하고 옆에 :8081 을 입력하면 미리 루트를 설정한 80번 포트의 index.html 화면이 뜸.

Dynamic & Static IP address(유동 IP와 고정 IP)

집(59.6.66.238) - ISP - 외부 사용자

ISP
개인이나 기업체에게 인터넷 접속 서비스, 웹사이트 구축 및 웹호스팅 서비스 등을 제공하는 회사를 말한다. 대표적으로 한국에서는 KT, SK브로드밴드, LG U+ 등이 있다.
외부 사용자가 집에 해당하는 IP주소를 치면 ISP를 통해 집 컴퓨터에 접속

만약, 외부 사용자가 많아진다면?

Dynamic IP address

  • 유동 IP를 사용.(나의 IP가 계속해서 바뀜. 속된 말로 돌려막기..)
  • 내가 만약 집을 비웠을 때? -> ISP가 내 IP주소를 회수함 -> 새로운 외부 사용자에게 나의 주소를 줌 -> 시간이 지나고 내가 집에 돌아오면? -> ISP가 나에게 새로운 IP를 부여(결국 나의 IP는 자꾸 바뀌게 됨[동적IP address]
  • 문제는? 자신의 컴퓨터에 웹 서버를 설치했을 때 -> 외부 사용자 접속 -> 내가 집을(웹 서버) 비우면서 ISP는 내가 다시 돌아왔을 때 새로운 IP를 부여 -> 외부 사용자가 다시 접속을 하려할 때 접속하는 IP는? -> 처음 외부접속자에게 알려줬던 옛날에 사용하던 IP 59.6.66.238 에 접속하게됨.(Dynamic IP address의 단점!!)

Static IP address

내 컴퓨터의 IP를 고정시켜 버림.
통신사(ISP)에게 일정 금액을 지불하면 통신사는 내 컴퓨터의 독점적 고정IP를 부여함.

DHCP(Dynamic Host Configuration Protocol)

    DHCP
  • 컴퓨터를 새로 사서 공유기에 연결(기존의 연결되어 있는 장치들과는 다른 IP번호를 직접 등록해야함. 원래는! )
  • 하지만, 연결하는 순간 자동으로 입력되도록 되어있음 -> 그것을 가능하게 해주는게 DHCP
  • 공유기는 DHCP Server 를 내장하고 있음. 또한, 인터넷을 사용하는 컴퓨터들에는 DHCP Client라는 프로그램을 기본적으로 내장하고있음.
  • 연결 과정: DHCP Cllient에서 DHCP Server에게 IP주소를 요청함 ->DHCP Server는 이용가능한 IP를 임대해줄 수 있다고 알려줌 -> Client는 그 IP를 사용하겠다고 응답함 -> Server는 Client가 빌려갔다는 사실을 서버에 저장함 -> 그리고 Client에게 "예 알겠습니다. 이 IP address는 2시간 동안 임대해드리겠습니다" -> Client는 Server가 준 정보를 받고 받은 정보를 등록함
  • DHCP 덕분에 모바일 혁명이 가속화 됐다.
